# Contact

Get in touch with the CAC Ontology community and project maintainers.

## Email Support

For questions about CAC Ontology or to join the CAC Ontology effort, please send an email to:

**support@projectvic.zendesk.com**

This is the primary contact method for:
- Joining the CAC Ontology effort
- Asking questions about the ontology
- General inquiries about the project
- Community participation

## Contributing

### How to Contribute

1. **Code Contributions**: 
   - Fork the repository
   - Create a feature branch
   - Submit a pull request
   - See [CONTRIBUTING.md](https://github.com/Project-VIC-International/CAC-Ontology/blob/main/CONTRIBUTING.md) for details

2. **Documentation**:
   - Improve existing documentation
   - Add examples and tutorials
   - Report documentation issues

3. **Testing**:
   - Test new features and modules
   - Report bugs and issues
   - Validate ontology files

4. **Community Support**:
   - Help answer questions
   - Share use cases and examples
   - Participate in discussions

## Reporting Issues

### Bug Reports

When reporting bugs, please include:

- Description of the issue
- Steps to reproduce
- Expected vs. actual behavior
- Environment details (tools, versions)
- Relevant ontology files or examples

### Feature Requests

For feature requests, please provide:

- Use case description
- Proposed solution or approach
- Benefits and impact
- Examples or references

## Security

For security-related concerns, please:

- **Do not** open public issues for security vulnerabilities
- Contact the project maintainers directly
- Follow responsible disclosure practices
